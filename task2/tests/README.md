# Архитектура E2E тестов

## Структура проекта

```
tests/
├── e2e/                    # End-to-end тесты
│   └── issues.spec.ts      # Основные тесты для работы с задачами
├── page-objects/           # Page Object Model
│   ├── IssuesPage.ts       # Page Object для страницы задач
│   └── BoardPage.ts        # Page Object для доски проекта
└── README.md               # Документация
```

## Page Object Model (POM)

### IssuesPage
Инкапсулирует все взаимодействия со страницей задач:
- Создание задач
- Поиск задач
- Открытие карточек задач
- Фильтрация по статусу
- Навигация на доску проекта

### BoardPage
Инкапсулирует взаимодействия с доской проекта:
- Проверка наличия колонок (To Do, In Progress, Done)

## Тест-кейсы

1. **Создание задачи** - проверяет создание новой задачи с заголовком и описанием
2. **Открытие карточки задачи** - проверяет открытие модального окна с деталями задачи
3. **Поиск задачи** - проверяет функциональность поиска задач
4. **Переход на доску проекта** - проверяет навигацию и наличие колонок на доске
5. **Фильтрация по статусу** - проверяет фильтрацию задач по различным статусам

## Запуск тестов

```bash
# Запуск всех тестов
npm test

# Запуск с UI
npm run test:ui

# Запуск в режиме отладки
npm run test:debug

# Запуск с видимым браузером
npm run test:headed

# Просмотр отчета
npm run test:report
```
